§1 · 系統組成

Overview — 系統組成

本系統由四層結構組成。使用者僅需操作 Inspector 與 EditorWindow，其餘各層為自動運作。

層級名稱職責
層 1 · Shader 本體Spine-Skeleton.shader主 Pass + Outline Pass。15 個 shader_feature 可被打包剝離
層 2 · InspectorSpineCustomShaderGUI取代 Unity 預設 Material Inspector 的自製介面
層 3 · RuntimeManager · Receiver · Lights場景光收集、SH 取樣、MPB 推送、自動掛載
層 4 · PresetProfile · Animator · HelperScriptableObject 預設檔、曲線動畫、程式 API
層 5 · EditorWindow7 套說明 / 管理視窗美術指南、程式指南、命名規則、平台管理、3 套元件 Help
層 6 · BuildBuildConfig · BuildPipeline平台 ScriptableObject + IPreprocessShaders 變體剝離
層 7 · DebugLightingIssueTroubleshooter八項自動診斷 + 一鍵修正 + 物件 Ping 跳轉
層 8 · 文件本說明書每章皆對應上述某個層級

命名空間：VFXTool.SpineSkeletonShaderTool

§2 · GUI 分區地圖

Architecture — GUI 分區總覽

Inspector 由六個可摺疊分區構成。點擊任一分區色帶可摺疊／展開；右鍵色帶可複製整區設定／貼上／重置。

每個子模組標題（如 Native Lighting Settings）右側皆有 P 圖示，展開後即為該子模組的 Preset 面板。詳細參數見 §6–§11。

分區子模組功能摘要
§6 MAINMain & Shadow Settings · Post-Processing Control主紋理、Tiling/Offset、主色、Cutoff、Straight Alpha、Bloom 抑制門檻同步
§7 NOISEGlobal Noise SettingsUV 動態雜訊 — 提供 BackLight / Inline / Outline 共用的隨機遮罩
§8 LIGHTINGNative · ShadowLight Base · NormalMap · ProbeUnity Realtime 燈、自製 ShadowLight、法線貼圖、Light Probe SH
§9 EMISSIONEmission Settings獨立 Emission 貼圖 + HDR 顏色 + Multiply with MainTex 選項
§10 EFFECTOverlay · Hit Sweep · BackLight整體染色、受擊光帶、背光高光 — 三組可獨立啟用
§11 LINEInline · Outline內描邊（Rim 風格）、外描邊（Sobel 8 鄰域）— 共用 Light Source 結構
共 12 個 shader_feature 模組，由 SpineCustomShaderGUI.OnGUI() 管理。
§3 · 視覺符號

▬

▏

§4 · 右鍵 · 頂部按鈕

兩個入口 — 找到任一屬性的說明

所有自製 Inspector 皆內建統一的查詢介面。當你不確定某個欄位的用途、依賴條件或排錯方式時，使用以下兩個入口。

① 右鍵任一屬性

對 Inspector 上任何一個屬性欄位按下滑鼠右鍵，會跳出 GenericMenu，選單中含「📖 查看使用說明」項目。點擊後系統會：

  • 開啟對應的 EditorWindow（美術指南 / 元件 Help）
  • 自動展開包含該屬性的章節
  • 自動捲動到該屬性卡片並以 Ping 黃色閃爍 2 秒

② 頂部 ▤ 使用指南按鈕

每個 Inspector 頂部都會繪製一條紅色／藍色／黃色橫條，標題為「▤ XXX 使用指南」。點擊可開啟對應的完整 EditorWindow。

每個 EditorWindow 都包含：

  • 頂部搜尋框 — 可搜尋屬性名／關鍵字
  • 可摺疊的模組分區（按色條展開）
  • 每個屬性附「依賴條件」與「排錯檢查」標籤
設計理念 — 任何在 Inspector 看到的欄位都應該能在 30 秒內找到對應說明。若某個屬性沒有右鍵說明或頂部按鈕無法開啟，請通報維護者。

實作：DrawPropWithContextMenu + DrawTopHelpButton（每個 Inspector 共用）

§6 · MAIN — Category 01/06

MAIN — 主紋理與基礎參數

本分區為所有材質必要設定，無法停用。包含 Main & Shadow Settings 與 Post-Processing Control 兩個子模組，共 7 個屬性。

自動掛載：Suppress Bloom 開啟 → 自動掛載 SpineBloomThresholdSync。前置條件：無

Module 01 · Main & Shadow Settings

系統會自動偵測同名後綴貼圖（_Normal / _BackLight / _Emission）並掛載。將材質拖到 SpriteRenderer 時亦會自動同步 MainTex 與 Straight Alpha 設定。

屬性名稱Shader 變數說明
Main Texture_MainTex主紋理。拖材質到 SpriteRenderer 時系統會自動讀取 Sprite 貼圖並開啟 Straight Alpha
Shared Tiling & Offset_MainTex_ST共用 UV (XY=Tiling, ZW=Offset)。此值會同步影響 MainTex、Emission、BackLight、Normal Map 確保所有疊加層對齊
Main Color (RGBA)_Color基礎顏色與全域透明度。Alpha 若小於 1 會連同 BackLight / Inline / Outline 一起變透明。Alpha 為 0 時若出現鬼影，多為 Beautify 壓暗或 Straight Alpha 未勾選所致
Shadow Alpha Cutoff_Cutoff · 0–1Shadow Pass 的 Alpha 裁切閾值；小於此值的像素不投射陰影
Straight Alpha Texture_StraightAlphaInput切換貼圖透明度計算模式。Spine 預設為 PMA；若邊緣出現黑邊或白邊請切換此項
自動偵測 — 若主紋理同目錄有 {name}_Normal / _BackLight / _Emission 後綴，系統在材質選取時自動填入。
注意 — Reset 整顆材質會清除所有 Preset 綁定與自動掛載元件。

Module 02 · Post-Processing Control

後製管線（如 WinkingBeautify）的 Bloom 經常將 HDR 描邊與發光過度過曝。本模組讓 Shader 取得目前後製 Bloom 強度，並依 AnimationCurve 反向換算出本材質專屬的抑制門檻。

屬性名稱Shader 變數說明
Suppress Bloom_EnableBloomSuppression啟用後系統自動掛載 SpineBloomThresholdSync 元件至角色。Layer 須正確
Bloom Threshold_BloomSuppressThreshold當前計算出的抑制門檻 — 由 SpineBloomThresholdSync 自動填入，唯讀，不需手動修改

SpineBloomThresholdSync 運作原理

  • 資料來源：Reflection 抓取 WinkingBeautify.finalBloomIntensity
  • 預設曲線：Bloom = 1.0 → Threshold = 0.6000；Bloom = 10.0 → Threshold = 0.5209
  • 寫入方式：MaterialPropertyBlock（不污染材質球）
  • 效能策略：髒標記（lastAppliedBloom）— 數值未變則跳過寫入
排錯 — 若 Suppress Bloom 勾選後仍過曝，請確認角色 Layer ∈ {8, 9, 12, 13, 18, 20}。Layer 不正確時自動化機制會將 BloomThresholdSync 元件移除。
§7 · NOISE — Category 02/06

NOISE — 全域雜訊取樣

本模組不直接影響畫面 — 它是供 BackLight、Inline、Outline 等模組選用的「亂度遮罩」來源。三者讀取同一張動態雜訊，因此節奏、速度、方向保持一致。

當其他模組啟用 Noise Mask 時，本分區自動連動開啟。Shader Feature：_MODULE_GLOBAL_NOISE

屬性名稱Shader 變數說明
Enable Global Noise_EnableGlobalNoise啟用 Shader 全域噪點運算，可用於消融、火焰外框等動態效果。啟用後需於子模組勾選 Noise Mask
Noise Scale (X, Y)_GlobalNoiseScaleUV 平鋪倍率。值越大顆粒越細密；建議 X 與 Y 同步調整
Noise Speed (X, Y)_GlobalNoiseSpeedUV 流動速度。若感覺沒在流動，請將此值調大
設計守則 — Noise 為共用資源。若你在多個模組同時啟用 Noise，預期它們會同節奏變化；需要差異感請改用各模組獨立的 Intensity 調幅。
§8 · LIGHTING — Category 03/06

LIGHTING — 光照系統

本章依序介紹四個子模組：Native Lighting（Unity Realtime）、Shadow Light Base（自製 SpineShadowLight）、Normal Map（法線取樣）、Light Probe（環境光 SH 取樣）。

前置條件：角色 Layer ∈ {8, 9, 12, 13, 18, 20}。啟用任一子模組 → 自動掛載 SpineLightReceiver；EnsureInstance 建立 SpineLightManager（場景單例）。

Module 01 · Native Lighting Settings

接收 Unity 場景的 Realtime Directional / Point / Spot Light。所有燈光資料由 SpineLightManager 依距離排序取前 8 盞，透過 MaterialPropertyBlock 推送。

屬性名稱Shader 變數說明
Enable Native Lighting_EnableNativeLighting啟用接收 Unity Realtime Light；關閉後本模組所有屬性失效。角色全黑時請確認 Master Intensity 不為 0
Master Light Intensity_NativeLightIntensity · 0–5所有 Realtime 光的總強度倍率 — 用於整體角色亮度調節
Directional Intensity_DirectionalLightIntensity場景中 Directional Light（太陽光）的獨立倍率
Additional Intensity_AdditionalLightIntensity場景中 Point / Spot Light 等附加光源的倍率，與 Directional 分開控制
Light Affects Additive_LightAffectsAdditive是否讓光照影響 Spine Additive Slot；預設關閉避免粒子過曝
Probe → Native Lighting_LightProbeNativeBlend · 0–1Light Probe 取樣值疊加到 Native Lighting 結果的比重；需先啟用 Light Probe。讓暗部不至於死黑

Module 02 · Shadow Light Base Settings

接收所有 SpineShadowLight 元件投射的自製光照 — 區域光、聚光、Cookie 投影、Caustics 波光。

屬性名稱Shader 變數說明
Enable Shadow Light Base_EnableShadowLight影響 BaseColor 的基底光照層（不含描邊與背光）
Base Intensity_ShadowLightBaseIntensity · 0–5ShadowLight 在 BaseColor 上的整體強度乘數
Affects Normal Map_SL_AffectsNormalMap是否參與 Normal Map 計算 — 開啟後光源方向會推算法線著色。需開啟 Normal Map

Module 03 · Normal Map Settings

讓 2D Sprite 接收方向性光照。自動偵測同目錄的 _Normal 後綴貼圖並掛載。

屬性名稱Shader 變數說明
Enable Normal Map Module_EnableNormalMap啟用法線採樣與光照計算
Normal Map_NormalMap法線貼圖。若主紋理同目錄有 {name}_Normal，會自動填入
Invert Normal Height_InvertNormalMap反轉法線 Z 軸高度（凹凸方向）
Normal Intensity_NormalIntensity · 0–5法線強度。0 為平面、1 為原始、5 為極度誇張
協同運作 — Normal Map 啟用後會被 Native Lighting、Light Probe、ShadowLight（若 Affects Normal Map = ON）三者同時取用。一張法線貼圖即可撐起整個立體感。

Module 04 · Light Probe Settings

取樣場景烘焙好的 Light Probe Group，把 9 個 SH 係數傳入 Shader。若場景開啟 LightMap + LightProbe 時此選項自動開啟。

屬性名稱Shader 變數說明
Enable Light Probe_EnableLightProbe總開關。BackLight / Inline / Outline 若選 LightProbe 光源，本選項必須啟用。若按鈕一直跳回關閉，代表場景缺乏 Lightmap / Probe 資料
Probe Intensity_LightProbeIntensity · 0–3SH 採樣的整體強度倍率

§8.5 · Light Source 多選旗標

BackLight、Inline、Outline 三個模組共用一套 Light Source 多選欄位（EnumFlagsField）。每勾選一個光源，下方展開該光源獨立的 Power / Color Blend / Intensity 三組旋鈕。

旗標光源類型Shader 變數
Flag 1 · RealtimeUnity 場景燈，前 8 盞_BackLightLightingMode / _InlineLightingMode / _OutlineLightingMode
Flag 2 · LightProbe烘焙環境光 SH（整數位元欄位）
Flag 4 · ShadowLight自製 SpineShadowLight—

進階區塊三件套

當 Light Source ≠ Disabled 時，BackLight / Inline / Outline 模組會額外展開：

  • A. Mask By Light Direction（方向遮罩）— 三光源各有 Power 指數，控制聚焦銳利度
  • B. Tint By Light Color（光源染色）— 三光源各有 Blend + Intensity 控制深度
  • C. Noise Mask（噪點侵蝕）— 需啟用 §7 Global Noise
選了 LightProbe 卻沒開全域 Light Probe — Inspector 會自動跳出修正按鈕。
§9 · EMISSION — Category 04/06

EMISSION — 自體發光

Emission 不接收任何光源 — 用於符文、機甲、武器尖端、Magic 物件等需要持續發光的區域。自動偵測 {name}_Emission 後綴貼圖。

屬性名稱Shader 變數說明
Enable Emission_EnableEmission啟用發光通道；關閉後完全跳過該段 Shader 計算
Emission Texture_EmissionTex指定發光部位遮罩（黑白或彩色）。自動偵測同目錄 {主貼圖名稱}_Emission
Emission Color (HDR)_EmissionColor疊加的發光顏色 — HDR 強度大於 1 會觸發後製 Bloom 過曝
Blend Weight_EmissionBlend · 0–1發光與底色的混合權重（0 = 不顯示，1 = 完全替換）
Intensity_EmissionIntensity · 0–10發光整體強度倍率。配合 HDR Color 推 Bloom
Multiply With MainTex_EmissionMulMain開啟後發光被主紋理乘調 — 讓符文受角色顏色影響；關閉則完全獨立
§10 · EFFECT — Category 05/06

EFFECT — 三個獨立特效模組

EFFECT 分區包含三個彼此獨立的特效模組：Effect Overlay（瞬間整體染色）、Hit Sweep（受擊光帶掃光）、BackLight（背光高光）。前兩者為時間性特效，BackLight 為常駐光照特效。

自動掛載：Hit Sweep → SpineHitSweepEffect 元件。本分區無 Toggle — Effect Intensity = 0 時自動視為關閉。

Module 01 · Effect Overlay — 整體色彩疊加

將指定 HDR 顏色按 Intensity 線性疊到最終畫面色。瞬間性特效 — 用於受擊閃白、變身、無敵、燃燒、冰凍。

屬性名稱Shader 變數說明
Effect Color (HDR)_EffectColor受擊閃白用 (5,5,5)；火焰用 (3,1.2,0.4)。若由程式 Runtime 控制，Preset 請取消勾選此項
Effect Intensity_EffectIntensity0–1 疊加比例。常配合 Curve 做 0→1→0 三幀閃光

Module 02 · Hit Sweep Settings — 受擊掃光

「打擊閃光」特效。傳入受擊點世界座標後，光帶從受擊點向外掃過整個角色。常用於武器命中、技能擊發、無敵讀條的視覺反饋。Shader 屬性在 Material Inspector 設定外觀；觸發則由 SpineHitSweepEffect 元件腳本控制（詳見 §13）。

屬性名稱Shader 變數說明
Enable Hit Sweep_EnableHitSweep啟用打擊光帶；同時系統自動掛載 SpineHitSweepEffect 至角色。角色 Layer 須正確
Sweep Color (HDR)_SweepColor光帶顏色。預設 HDR 白光 (2,2,2) 觸發 Bloom 過曝產生亮邊
Sweep Width_SweepWidth · 0.01–1光帶寬度，以角色直徑比例計算。0.15 為標準，0.3 較粗
Sweep Softness_SweepSoftness · 0.001–1光帶邊緣的羽化柔和度，數值越大邊緣越糊
進階控制屬性（位於 SpineHitSweepEffect 元件 Inspector 而非 Material）：duration（掃光總時間，秒）、sweepRange（預設 -4 → 4）、sweepCenterOffset（角色中心校正向量）。詳見 §13。

Module 03 · BackLight Settings — 背光高光（基底）

背光／高光通道。BackLight 是 EFFECT 分區中最複雜的模組 — 接受三種光源並產生獨立的 Power / Blend / Intensity 旋鈕集合。自動偵測 {name}_BackLight 後綴貼圖並掛載。

屬性名稱Shader 變數說明

§16 · Runtime 05

其他三個自動管理的元件

SpineAlphaMaskRenderer · SpineBloomThresholdSync · SpineEffectBootstrapper — 這三個元件不會在角色 Inspector 上被直接操作，由 SpineCustomShaderGUI 在屬性變動時自動掛載／移除。

共同前提 — 自動掛載僅對 Layer ∈ {8, 9, 12, 13, 18, 20} 的物件作用。Validate 機制會在每次選取時掃描；若 Layer 不正確會把元件移除並在 Inspector 頂部跳出醒目警告。

SpineAlphaMaskRenderer

觸發：Enable Inline 勾選

使用 CommandBuffer 在主相機 BeforeForwardOpaque 時繪製一張 RenderTexture（Alpha Mask）供 Inline Shader 取樣。

  • resolutionScale：RT 解析度倍率，預設 0.75
  • 綁定相機：Camera.main，場景切換時自動重新綁定

SpineBloomThresholdSync

觸發：Suppress Bloom 勾選

透過 Reflection 抓取後製 Beautify 的 finalBloomIntensity，依 AnimationCurve 推算 _BloomSuppressThreshold 寫入 MPB。

  • 髒標記：lastAppliedBloom 未變則跳過寫入
  • 預設曲線：1.0 → 0.60；10.0 → 0.52

SpineEffectBootstrapper

手動使用（程式 API）

執行階段一次性套用一份 Profile，並自動補齊所有依賴元件（Receiver / AlphaMask / HitSweep / BloomSync）。適合角色切換特效時引入完整效果包。

  • ApplyProfile()：套用 + 補齊依賴
  • ClearAllEffects()：一次清除所有 Spine 特效

§17 · Preset 01

Preset 系統概念

把模組設定存成資產 — 跨材質球共用。每個子模組皆可獨立儲存為一個 ScriptableObject（SpineModuleProfile.asset）。

掛載後 Inspector 中該模組的屬性會顯示 ✦ 符號，並由 SpineProfileAnimator 透過 MaterialPropertyBlock 即時驅動 — 不寫入材質球本體，多個角色可共用。

操作步驟（Inspector 端）

  1. 點 P 按鈕展開 Preset 面板（每個子模組標題右側皆有 P 圖示）
  2. 設定 Save Mask：勾選想保存的屬性 — 未勾選的不會寫入 Preset
  3. 按 Save 儲存為 .asset 檔
  4. 掛到角色的 SpineProfileAnimator，Inspector 顯示 ✦ 即生效

關鍵設計原則

原則說明
儲存類型Float / Color (HDR) / Vector4 / Texture
寫入機制可選擇靜態值，或開啟 ∿ 切到 AnimationCurve 驅動
忽略名單ProfileIgnoredProperties — Enable Toggle 與 LightingMode 不入 Preset，避免動畫切換模組
自動防呆路由將 Inline 的 Preset 拖到 Outline 區？系統會偵測 moduleName 並自動導向正確分區

§18 · Preset 02

SpineModuleProfileEditor — 全場景自動同步

在 Project 視窗中直接點選一個 .asset 時，由 SpineModuleProfileEditor 接管 Inspector。修改 Profile 的同時，會自動掃描全場景所有套用此 Profile 的角色，刷新自動掛載狀態。

編輯器特性

功能說明
頂部色帶依 moduleName 自動著色 — 一眼看出這份 Profile 是哪個分區

  3. Profile > Material 雙重驗證 — 先查 Profile 中是否定義該功能；無則查 Material — 決定是否需自動掛載

§19 · Save Mask · §20 · AnimationCurve

Save Mask 與 AnimationCurve 驅動

精確控制哪些屬性由 Preset 管理、哪些留給程式自由控制；並支援逐屬性的曲線動畫引擎。

§19 · Save Mask — 參數遮罩

優先權：MaterialPropertyBlock（最高）> Material Instance > Shared Material

若某個參數需由程式在 Update 中「每幀高頻控制」（如閃爍、變色），請在 Save Properties 中取消勾選該參數。如此 Profile 不會綁定該參數，程式可安全用 SpineProfileHelper.SetFloat 直接修改 MPB，不會被 Animator 覆寫。

常見錯誤 — 把 Effect Color 或 Fill Inside 等需要程式控制的屬性寫入 Preset，導致受擊閃白功能無效。

§20 · AnimationCurve 驅動

每個 ✦ 屬性右側都有 ∿ 按鈕。點開後該值改由 AnimationCurve 取代靜態值；時間軸從 GameObject 生命週期開始，可由 Animator.Restart() 重置。

屬性類型曲線數量說明
Float一條 curveX控制 0–1 數值
Color四條曲線R/G/B/A 各自驅動
Vector4四條曲線X/Y/Z/W 各自驅動
Texture不支援僅靜態值
被忽略的屬性 — Profile 不儲存 Enable 類 Toggle 與 LightingMode（避免動畫意外切換整個模組）。詳見 ProfileIgnoredProperties 清單。

SpineProfileAnimator 每幀讀取 Profile + 評估曲線 + 寫入 MPB

§21 · Preset 04

SpineNamingRuleWindow — 預設檔命名規則

兩種命名格式 — 角色特化與通用型。所有 Preset 預設檔都應遵循下列其中一種命名格式。

格式：{角色英文代稱}_{模塊名稱}_{序號}

§22 · Preset 05

SpineProfileHelper API

強烈建議使用 SpineProfileHelper 靜態類別控制 Profile，而非自行操作 SpineProfileAnimator 元件。Helper 內建生命週期防錯、重複掛載偵測與效能自動回收。

三個高階 API（推薦）

// 1. 動態加載 / 替換 Profile
//    自動掛載 Animator、防重複、強制刷新
SpineProfileHelper.ApplyProfileRuntime(myRenderer, newProfile);

// 2. 移除指定模塊 (Enum 避免拼錯)
SpineProfileHelper.RemoveProfileRuntime(
    myRenderer,
    SpineProfileModule.Outline);

// 3. 一鍵清除角色身上所有 Profile 與緩存
SpineProfileHelper.ClearAllProfiles(myRenderer);

使用 Enum 取代字串避免拼錯：

§23 · Preset 06

ApplyTempProfile — 狀態切換的最佳解

當角色進入特殊狀態（中毒、冰凍、變身）時，不建議直接修改 MPB — 因為會被曲線蓋掉。最佳解：呼叫 ApplyTempProfile，系統會複製一份 Profile，將指定參數強制覆寫，並關閉該參數的曲線動畫。

程式範例

// 1. 準備參數字典
var overrides = new Dictionary<string, object> {
    { "_OutlineColor", Color.green },
    { "_OutlineWidth", 5.0f }
};

// 2. 套用副本，回傳實體存起來
SpineModuleProfile activeTempProfile
    = SpineProfileHelper.ApplyTempProfile(
        myRenderer, baseProfile, overrides);

// 3. 狀態結束時，移除效果並銷毀副本
SpineProfileHelper.RemoveProfileRuntime(
    myRenderer, activeTempProfile.moduleName);
Destroy(activeTempProfile);
記憶體洩漏警告 — ApplyTempProfile 回傳的 TempProfile 實體必須手動 Destroy。否則每次狀態切換都會在記憶體中產生新的 ScriptableObject 殘留。

底層原理

Temp Profile 為原 Profile 的深拷貝，在 overrides 中指定的屬性會被覆寫且 useCurve = false（停用曲線）。原 Profile 不會被修改。

典型使用場景

  • 受擊瞬間將 Outline 變紅
  • 變身時將整個角色 Emission 染色
  • 無敵狀態下將描邊閃光放大

SpineProfileHelpWindow → 「進階：Temp Profile 實體竄改」章節

§24 · Preset 07 · §24.1 · Preset 08

底層 MPB 安全 API · SpineProfileTester

當某個參數在 Save Mask 中未被勾選時，使用 Helper 封裝的靜態 API 安全寫入 MPB，與系統底層 MPB 狀態無縫融合，不打架，並保持極致合批效能。

§24 · 正確與錯誤寫法對照

寫法說明
✓ 正確SpineProfileHelper.SetFloat(myRenderer, "_OutlineWidth", 0.1f);無材質實例化、不污染 sharedMaterial、不打斷 SRP Batching
✓ 正確SpineProfileHelper.SetColor(myRenderer, "_EffectColor", Color.red);同上
✓ 正確SpineProfileHelper.SetVector(myRenderer, "_GlobalNoiseSpeed", new Vector4(1,1,0,0));同上
✗ 錯誤myRenderer.material.SetFloat("_OutlineWidth", 0.1f);會產生材質實例，可能被 Animator 覆寫，打斷 SRP Batching
✗ 錯誤myRenderer.sharedMaterial.SetFloat("_OutlineWidth", 0.1f);修改所有共享此材質的角色！
前提 — 必須確保該參數在 ShaderGUI 儲存 Preset 時沒被勾選。否則 Animator 會把你寫入的值覆蓋掉。

§24.1 · SpineProfileTester — 互動式測試工具

在 Hierarchy 內加入此測試元件後進入 Play Mode，可用滑鼠快速驗證 Profile 系統的完整流程。

滑鼠操作觸發行為驗證目的
滑鼠左鍵在指定 Renderer 上動態套用 / 移除指定的 Profile 資產驗證 ApplyProfileRuntime 與 RemoveProfileRuntime
滑鼠右鍵直接修改指定 _PropertyName 的 MPB 數值（不經 Profile）驗證 Save Mask 未勾選屬性的程式控制是否正常
滑鼠中鍵呼叫 ApplyTempProfile 產生竄改副本驗證 §23 的臨時覆寫流程
建議流程 — 新建 Profile 後 → 拖入測試器 → Play → 左鍵套用 → 右鍵竄改未勾選屬性 → 中鍵測試臨時 Profile → 三項都正常即可正式接入專案戰鬥邏輯。

§26 · Build 02

SpineShaderBuildConfig — 雙重鎖驗證

系統提供兩種運作模式 — 編輯器中所有 Keyword 強制全開以供美術預覽（Editor Default）；切換到平台 Config 時則以「雙重鎖」決定材質球的 Keyword 啟用狀態。

A · Editor Default 模式

執行 SpineShaderBuildPipeline.SyncAllEnabled() 後，會強制掃描全專案的 Spine 材質球，將所有 15 個 Module Keyword 一律 EnableKeyword。SessionState 記為 EditorMode = true。

適合美術做設計時 — 不論勾不勾，都能看到效果。不影響正式打包效能。

B · Platform Config 模式（雙重鎖）

執行 SyncForPlatformConfig() 後改用雙重驗證：

驗證條件說明
isAllowedByConfig該 Keyword 在平台 Config 中是否被勾選
isToggledByUser材質球中對應 _Enable* 屬性是否為 ON
最終結果只有「兩者都成立」時才 EnableKeyword

第三道過濾 — OnProcessShader（打包時）

IPreprocessShaders 介面在編譯每個 Variant 時，會檢查兩件事：

  1. 平台 Config 是否放行此 Keyword
  2. 全專案是否有任一材質球實際用到此 Keyword

任一不符即將該 Variant 從 IList<ShaderCompilerData> 中移除。Outline / Inline 等重 Pass 在不需要時完全不會編譯。

Config 路徑：Assets/Scripts/VFXTool/BuildConfigs/SpineSkeletonShaderTool/

§27 · Build 03

SpineShaderBuildPipeline — 程式 API

為打包自動化腳本使用。若你的專案有自動化打包流程（CI / 一鍵打包按鈕），可透過 SpineShaderBuildPipeline 的靜態方法在 OnPreprocessBuild 階段刷新材質球設定。

完整 API 清單

方法說明
SyncAllEnabled()恢復 Editor Default 全效預覽模式，強制開啟所有 15 個 Keyword
SyncForPlatformConfig(cfg)依指定 Config 套用平台雙重鎖
SyncKeywordsForTarget(name)傳入平台名稱（字串）刷新全專案材質球 — CI 用
IsEditorDefaultMode()查詢當前是否為 Editor Default 模式
OnPreprocessBuild自動於打包前掃描，按 BuildTarget 套用對應 Config

CI 使用範例

public class SpineBuildAutomation {
    public void PrepareForTarget() {
        // 例如切到 Android
        SpineShaderBuildPipeline
            .SyncKeywordsForTarget("Android");
    }
}
關於 MissingReferenceException — 若 Config 資產被外部刪除或遺失，PlatformManager 視窗會偵測並顯示「⚠️ 發生異常 (MissingReferenceException)」面板，提供「重新整理」按鈕讓你恢復視圖。

§30 · Debug 02

SpineDebugHelpWindow — 常見問題 Q&A

兩個最常出現的問題與標準解法。所有問題卡片均可在頂部搜尋框輸入關鍵字過濾。

Q1 · 光照與陰影問題

「為何角色看起來黑黑的，明明有開燈但沒受光照？」

標準排查順序（DebugHelpWindow 列出）：

  1. SpineLightManager 層級規範 — 該 Layer 是否被加入收集名單
  2. 確認材質球已勾選 Enable Native Lighting
  3. 確認 GameObject Layer ∈ {8, 9, 12, 13, 18, 20}
  4. 確認場景燈光的 Culling Mask 包含上述 Layer
  5. 按 ▤ 全效預覽按鈕 — 若恢復正常，代表是平台 Config 剝離了該模組

Q2 · 特效與模組缺失

「為何勾選內外框、背光、掃光都不起作用？」

標準解法：

  1. 再次確認 Layer ∈ {8, 9, 12, 13, 18, 20} — 許多特效依賴腳本抓取資料
  2. 按 ▤ 全效預覽 — 強制開啟此材質的所有 Shader 變體
  3. 若全效預覽下恢復正常，代表「平台變體管理」剝離了該 Keyword
本地完整預覽 — 全效預覽僅影響當前編輯器，不影響打包效能。隨時可開關。
§31 · Workflow

入門教學 — 從零建立一個受光照的 Spine 角色

六個步驟。本節為最短上手流程。任一步驟出問題請翻閱 §29 排查器章節。

  • 01
    建立材質球，Shader 選 Spine / Skeleton
    拖到 SpriteRenderer — 主紋理、Straight Alpha、_Normal / _BackLight / _Emission 系列檔自動就位

    角色 GameObject 設到 Layer 8（或 9 / 12 / 13 / 18 / 20）
    否則自動化機制會清除所有掛載元件。Inspector 頂部有一鍵修正按鈕
  • 03
    開啟需要的模組（例如 LIGHTING / Normal Map / Outline）
    SpineLightReceiver、AlphaMaskRenderer、LightManager 會在勾下 Toggle 時自動掛載
  • 04
    在場景放一盞 Realtime Directional Light

    需要動畫？將模組存成 Preset 並加入 ∿ 曲線
    SpineProfileAnimator 即時驅動。命名請遵循 §21 規則
  • 06
    打包前檢查 — 開啟 §25 SpinePlatformManagerWindow，切到目標平台
    確認該平台 Config 中啟用了你需要的所有 Module Keyword，按「套用 {平台名}」刷新材質球
